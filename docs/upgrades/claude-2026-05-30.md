### Experimental / recent features
- **Fast mode** — Opus-only, ~2.5× faster, identical quality, higher cost
  (`/fast` or `fastMode: true`; Opus 4.8/4.7/4.6). Good for tight iteration loops.
- Env flags worth trialing (all default 0, opt-in):
  - `CLAUDE_CODE_EFFORT_LEVEL` — pins effort (low/medium/high/xhigh/max/auto);
    precedence over `/effort` and `effortLevel`.
  - `EXPERIMENTAL_AGENT_TEAMS=1`, `FORK_SUBAGENT=1` (forked subagents inherit full
    context), `AUTO_BACKGROUND_TASKS=1` (auto-background long tasks after ~2 min).
- Informational: Dynamic Workflows (`/workflows`, v2.1.154 — already used in this
  repo), plugin auto-load from the skills dir + `plugin init` (v2.1.157), Auto mode
  on Bedrock/Vertex/Foundry (v2.1.158).

## Repo changes applied this run (wording-only, no behavior change)
1. Front-loaded the **Claude Code** anchor in the `khenrix-upgrade` descriptions
   (all 3 CLI variants) to reduce over-triggering on generic "upgrade my setup".
2. Made the `make khenrix-refresh` working directory explicit ("from the repo root,
   not the plugin dir") in all 3 `khenrix-upgrade` skills.
3. Hardened script-invocation paths in the codex/agy variants of both
   `khenrix-setup` and `khenrix-upgrade`: codex now uses `$PLUGIN_ROOT/…`, agy uses
   the fixed install path `~/.gemini/config/plugins/khenrix-utils/…` — dropping the
   fragile relative-cwd assumption (claude already used `${CLAUDE_PLUGIN_ROOT}`).

## Deferred live-config recommendations (apply yourself)

Not auto-applied (per the skill's design). Suggested, in priority order:

1. **Pin effort to xhigh for coding** (biggest lever):
   ```bash
   /effort xhigh                      # in-session
   export CLAUDE_CODE_EFFORT_LEVEL=xhigh   # persistent (shell profile)
   ```
2. **Pin Opus 4.8 explicitly** (optional; it's already the default):
   ```bash
   /model opus       # in-session, or set "model" in ~/.claude/settings.json
   ```
3. **Use Fast mode for Opus iteration loops** when latency matters:
   ```bash
   /fast             # toggle, or "fastMode": true in settings.json
   ```
4. **Trial experimental flags** (in your shell profile, then restart Claude Code):
   ```bash
   export EXPERIMENTAL_AGENT_TEAMS=1
   export FORK_SUBAGENT=1
   export AUTO_BACKGROUND_TASKS=1
   ```
